# Research Notes

This directory contains raw research notes, meeting minutes, and observations about Rust in the Linux Kernel.

## Purpose

Use this space for:
- **Meeting notes** from community discussions
- **Personal observations** while studying the codebase
- **Rough drafts** before they become formal documentation
- **Experimental ideas** and brainstorming
- **Learning journal** as you explore Rust-for-Linux

## Organization

Suggested structure:
```
notes/
├── YYYY-MM-DD-topic-name.md    # Date-prefixed notes
├── meetings/                    # Meeting minutes
├── personal/                    # Personal learning notes
└── ideas/                       # Experimental ideas
```

## Note-Taking Tips

1. **Date your notes**: Use YYYY-MM-DD format for easy sorting
2. **Be specific**: Include context and links to references
3. **Track sources**: Always cite where information came from
4. **Version control**: Git tracks history, so iterate freely
5. **Link related notes**: Cross-reference when topics overlap

## Example Note Structure

```markdown
# Topic Name - YYYY-MM-DD

## Context
What prompted this note?

## Observations
What did you learn or notice?

## Questions
What remains unclear?

## Next Steps
What should be investigated further?

## References
- [Link to source]
- [Related discussions]
```
